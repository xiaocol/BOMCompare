# BOM構成比較管理システム

React、Vite、React Flow を利用した、日本語 UI の BOM 構成比較プロトタイプです。
現行 BOM を基準データとして参照し、新 BOM を編集しながら、追加、削除、変更を画面上で確認できます。

## 概要

このシステムは、指定した `親品目番号` を最上位品目として、現行 BOM と新 BOM を左右に並べて表示します。
左側の `現行BOM` は参照専用、右側の `新BOM` は編集対象です。

新 BOM 側では、子品目の追加、品目情報の修正、ドラッグ操作による構成変更ができます。
変更内容は現行 BOM と自動比較され、ツリー図と一覽表に `追加`、`削除`、`変更`、`両方` として表示されます。

## 主な機能

- 管理画面向けのサイドメニュー付きレイアウト
- サイドメニューの展開、収縮
- `親品目番号` を指定した BOM 構成の読み込み
- React Flow による左右 2 画面の BOM ツリー表示
- 現行 BOM の参照専用表示
- 新 BOM の子品目追加、品目修正
- 新 BOM のドラッグ操作による親子関係変更
- 新 BOM のドラッグ操作による構成間挿入
- 移動操作専用の戻しボタン
- 一般編集操作の戻しボタン
- CSV による現行 BOM 読み込み
- CSV による新 BOM 読み込み
- 新 BOM の CSV 出力
- 下部一覽表での差異確認
- ブラウザ localStorage による作業状態の自動保持

## BOM 比較ルール

比較は `親品目番号` で指定した最上位品目の配下に対して行います。

差異判定は、主に以下の項目を比較します。

- 親品目番号
- 子品目番号
- 品目名称
- 版数
- 数量
- 有効開始日
- 有効終了日
- 親子関係

表示されるステータスは以下です。

| 表示 | 意味 |
| --- | --- |
| 両方 | 現行 BOM と新 BOM の両方に存在し、主要項目に差異がない |
| 変更 | 現行 BOM と新 BOM の両方に存在するが、版数、数量、有効期間、親子関係などに差異がある |
| 追加 | 新 BOM にのみ存在する |
| 削除 | 現行 BOM にのみ存在する |

## 画面構成

### 現行BOM

左側に表示される基準 BOM です。
現行 BOM は参照専用であり、ノードの追加、修正、移動はできません。

### 新BOM

右側に表示される編集対象 BOM です。
新 BOM では以下の操作ができます。

- 子品目追加
- 品目修正
- ノードを別ノードへドラッグして親子関係を変更
- ノードを親子線の間へドラッグして中間構成として挿入
- 直前の移動操作を戻す

### BOM構成一覽

画面下部の一覽表では、現行 BOM と新 BOM の行を統合して表示します。
両方に存在する品目は 1 行にまとめられ、差異がある項目は `変更前 → 変更後` の形で表示されます。

## ドラッグ操作

新 BOM 側の子品目ノードはドラッグできます。

### ノード上へドロップ

あるノードを別のノード上へ移動すると、移動したノードは移動先ノードの子品目になります。

例:

```text
P-A-B
P-C
```

`B` を `C` の上へ移動すると、以下になります。

```text
P-A
P-C-B
```

### 親子線の間へドロップ

ノードを親子関係の線上へ移動すると、移動したノードが親子関係の間に挿入されます。

例:

```text
A-B
```

`C` を `A-B` の間へ移動すると、以下になります。

```text
A-C-B
```

### 無効な場所へドロップ

ノードが有効な移動先にドロップされなかった場合、ノードは自動的に元のレイアウト位置へ戻ります。

## CSV 形式

CSV の列形式は以下です。

```csv
亲品目,子品目,有效开始日,有效终了日,版数
PRD-9000,ASM-A100,2026-05-01,2099-12-31,C
ASM-A100,MOD-A110,2026-05-01,2099-12-31,B
```

各列の意味は以下です。

| 列名 | 内容 |
| --- | --- |
| 亲品目 | 親品目番号 |
| 子品目 | 子品目番号 |
| 有效开始日 | 有効開始日 |
| 有效终了日 | 有効終了日 |
| 版数 | 版数 |

CSV 読み込み時は、親子関係からツリー構造を自動生成します。

## 初期データ

初期デモデータは [bom-data.json](./bom-data.json) に格納されています。
このデータには、多層構造の BOM と、追加、削除、変更を確認できる差異データが含まれています。

ブラウザ上の編集内容は localStorage に保存されます。
初期データを再読み込みしたい場合は、localStorage の該当データを削除するか、アプリ側の保存キーを変更してください。

## セットアップ

Node.js と npm が利用できる環境で、以下を実行します。

```bash
npm install
```

## 開発サーバー起動

```bash
npm run dev
```

起動後、以下の URL をブラウザで開きます。

```text
http://127.0.0.1:5173/
```

## 本番ビルド

```bash
npm run build
```

ビルド結果は `dist/` に出力されます。

## プロジェクト構成

```text
BOM/
  index.html

  vite.config.js
  bom-data.json
  src/
    main.jsx

```

## 注意事項

- 現行 BOM は参照専用です。
- 編集対象は新 BOM です。
- CSV 出力は新 BOM の構成を対象にしています。
- CSV 読み込みは、現行 BOM と新 BOM のどちらにも個別に実行できます。
- このアプリはプロトタイプであり、サーバー保存やユーザー認証は未実装です。
